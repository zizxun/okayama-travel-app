# 手機使用指南

這個旅行 App 目前是純靜態網站，不需要後端。手機上有三種用法，推薦程度如下。

## 推薦：部署成 HTTPS 網頁後加入主畫面

適合正式旅行時使用。把整個資料夾上傳到 GitHub Pages、Netlify、Cloudflare Pages 或任何 HTTPS 靜態網站空間。

上傳後用手機開啟網址：

- iPhone Safari：分享按鈕 -> 加入主畫面。
- Android Chrome：選單 -> 加到主畫面 / 安裝應用程式。

部署成 HTTPS 後，`service-worker.js` 會啟用離線快取。第一次打開並載入完成後，之後沒網路也能開基本行程、點位、預算和清單。

## 可用：把整個資料夾複製到手機

把這些檔案保持在同一個資料夾：

- `index.html`
- `styles.css`
- `app.js`
- `manifest.webmanifest`
- `service-worker.js`
- `icons/okayama-icon.svg`
- `icons/okayama-icon-192.png`
- `icons/okayama-icon-512.png`

然後在手機檔案 App 或瀏覽器開 `index.html`。這種方式可以看行程與互動，但通常不能安裝成真正 PWA，也不一定能啟用離線快取。

## 測試：同 Wi-Fi 用電腦開本機伺服器

在旅行前測試手機畫面可用。電腦和手機連同一個 Wi-Fi，於 `D:\codex\岡山` 啟動靜態伺服器後，用手機開電腦的區網 IP。

例如網址會長得像：

```text
http://192.168.x.x:8000/
```

這種方式適合測試，不適合旅行正式使用，因為電腦關掉後手機就開不到。

## 旅行前檢查

- 手機瀏覽器能打開首頁。
- 每日行程切換正常。
- 行程頁的「今日地圖」可以開啟當日整體路線。
- 行程頁的「景點地圖」可以逐一開啟 Google Maps。
- 點位的地圖連結能開 Google Maps。
- 行前清單勾選後，重新整理仍保留狀態。
- 修改每日行程時間、新增或刪除項目後，重新整理仍保留狀態。
- Trip 區塊的備忘錄重新整理後仍保留。
- 若已部署 HTTPS，開飛航模式後仍能重新打開 App。

## 關於本機儲存

App 裡的行程修改會存在目前這支手機的瀏覽器資料裡，不會寫回 GitHub Pages，也不會自動同步到其他人的手機。

目前會存在本機的內容：

- 每日行程時間與內容修改。
- 新增/刪除/排序後的行程項目。
- 新增/刪除/排序後的餐廳、景點或備案。
- 行前清單勾選狀態。
- Trip 區塊的備忘錄。

iPhone 刪掉主畫面的 App 通常會移除那個 PWA 入口，但網站資料與快取是否完全清掉會依 iOS/Safari 版本而不同。旅行結束若想保險，先在 App 的清單頁按「清除本機資料」，再刪掉主畫面 App。

## 關於多人同步

如果把修改「寫回 GitHub」，大家重新載入 GitHub Pages 後理論上就能看到同一份資料。但純前端 App 不適合直接寫 GitHub，因為需要 GitHub token；把 token 放在手機網頁裡會被看見，安全性不好。

比較穩的同步做法：

- 簡單版：每個人本機修改，必要時用匯出/匯入 JSON 交換版本。
- 正式版：加登入與雲端資料庫，例如 Firebase、Supabase 或 Cloudflare。
- GitHub 版：做一個安全後端代替手機寫 GitHub，而不是讓手機直接帶 token。

## 如果要請 Codex 協助部署 GitHub Pages

需要準備其中一種方式：

- 你已經有 GitHub repo：提供 repo 名稱或網址，並確認目前這個資料夾可以推上去。
- 你要新建 repo：提供 repo 名稱、是否公開，以及 GitHub Pages 想用的網址。
- 如果使用 GitHub 連接器或 `gh` CLI：需要在 Codex 環境裡有授權，才能幫你 push 和設定 Pages。

不建議把 GitHub personal access token 寫進這個 App 或直接貼在前端檔案裡。
